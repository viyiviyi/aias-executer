## 工作原则

- **事实求是** 禁止输出或创建项目中不存在的虚假内容，一切以项目实际为准
- **实用主义** 实用且符合原项目规范优先，禁止过度设计
- **有计划有步骤** 有完整的计划和步骤，且用户确认可行后才实施

## 工具使用建议

- 终端和命令行工具均有只能查看最新100行输出的限制，不能用于获取较长的内容
- 读取文件时行范围控制在200行内，防止遇到超大文件，或者获取列表时查看行数和字符数，内容有限时一次性读取
- 优先更新文件内容而不是重新写入文件，更新时行号通过read_code获取，基于获取的行号操作，操作完成后可再次获取操作后的内容和行号
- 需要多次交互执行命令行时建议使用终端，如ssh访问服务器、编译、安装服务等操作
- 任一操作重试次数不能超过3次，达到重试次数后停止工作
- 绝对不能用宽泛或不明确的删除操作
- skills目录可能有相关的技能，如果新安装技能也安装在此目录

## 项目维护指南

1. 维护项目时需要先了解原项目功能和架构，以符合原项目架构的方式尽可能少改动的维护项目。
2. 项目开发文档按模块维护，目录默认维护在项目同级dev-readme.md文件，具体文档在dev-docs目录，文件名用中文，markdown格式，通过相对路径链接到具体文件
